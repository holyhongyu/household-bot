# CLAUDE.md

## Changelog rule
Whenever you complete a meaningful piece of work (a new feature, a fix, or
a significant change) in this project, append an entry to the TOP of
CHANGELOG.md (create it if it doesn't exist) in this format:

### [Today's date]
- Plain-English bullet describing what changed and why it matters — no code
  jargon, no function names, written for someone who isn't a programmer
- If something is still broken or incomplete, say so clearly

Keep entries short — 2-4 bullets max per session. Do this automatically,
without being asked each time.
